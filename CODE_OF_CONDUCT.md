# Contributing to TMF921 Intent Translator

We love your input! We want to make contributing to TMF921 Intent Translator as easy and transparent as possible.

## Development Process

1. Fork the repo and create your branch from `main`
2. If you've added code that should be tested, add tests
3. If you've changed APIs, update the documentation
4. Ensure the test suite passes
5. Make sure your code lints
6. Issue that pull request!

## Code Style

* Use consistent indentation (2 spaces for Python, configured by project for JS/TS)
* Write meaningful commit messages
* Comment your code where necessary
* Keep functions focused and small

## Pull Request Process

1. Update the README.md with details of changes if applicable
2. Update the docs/ if you added new features
3. The PR will be merged once you have the sign-off of at least one maintainer

## Reporting Bugs

Please use the GitHub Issues to report bugs with the following template:
- OS/Browser version
- Steps to reproduce
- Expected vs actual behavior
- Screenshots if applicable

## Suggesting Enhancements

We welcome suggestions! Please use GitHub Issues with the label "enhancement" and provide:
- Clear description of the feature
- Use case / motivation
- Potential implementation approaches
